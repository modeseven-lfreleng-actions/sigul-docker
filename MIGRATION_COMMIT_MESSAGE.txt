Migrate from python-nss to python-nss-ng v0.1.0

This commit completes the migration from the unmaintained python-nss library
to the modern python-nss-ng v0.1.0 package published on PyPI.

## Motivation

python-nss has been unmaintained for nearly 10 years and was designed for
Python 2.x. It has poor error reporting from NSS/NSPR C libraries, making
debugging TLS and certificate issues extremely difficult in our containerized
environment. python-nss-ng is a modernized replacement with:

- Python 3.10-3.14 support
- Improved error message propagation from NSS/NSPR to Python
- Modern build system and active maintenance
- NSS 3.117+ compatibility

## Changes Made

### Installation Method
- **Before**: Built python-nss from GitHub source (time-consuming, unreliable)
- **After**: Install python-nss-ng v0.1.0 from PyPI (fast, standard)
